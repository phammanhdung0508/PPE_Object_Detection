## 2025-05-15 - Accessible File Inputs and Interactive Toggles
**Learning:** Using `display: none` on interactive elements like file inputs completely removes them from the keyboard tab order, making them inaccessible to screen readers and keyboard users. Additionally, icon-only buttons or toggles that don't update their ARIA attributes (like `aria-pressed` or `aria-label`) provide a confusing experience for assistive technology users.
**Action:** Always use a `visually-hidden` CSS pattern for hidden interactive elements to keep them in the DOM and tab order. Use adjacent sibling selectors (`input:focus-visible + label`) to provide visual focus indicators for these hidden elements. For toggle actions (like Maximize/Restore), always dynamically update both the visible text and the associated ARIA attributes (`aria-label`, `aria-pressed`) to ensure the state is clear to all users.
